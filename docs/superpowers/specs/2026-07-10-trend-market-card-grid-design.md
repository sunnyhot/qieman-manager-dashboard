# 趋势报告市场卡片统一布局设计

## 背景

趋势报告的“市场视图”目前在周期判断和板块判断中都使用 `adaptive(minimum: 200)`。宽窗口会按可容纳的最大列数切分空间：只有三项的周期卡片集中在左侧，而板块卡片铺成更多列。同时卡片只约束宽度，高度随文案自然变化，因此出现明显的留白和高矮不齐。

## 已确认目标

- 保留卡片正文，不以固定高度截断信息。
- 同一行卡片按该行最高内容等高，形成稳定基线。
- 周期判断和板块判断使用相同的三列节奏，避免同一区域出现大小两套规格。
- 只调整市场视图卡片网格，不重排操作建议、核验或其他工作台界面。

## 方案

### 布局

周期卡片和板块卡片共用一组三个等宽的弹性列。应用当前最小窗口宽度为 860 点，三列在主内容区仍可维持可读宽度；宽屏时三列共同分配全部可用空间，不再产生周期卡片右侧的空列。

每个卡片在网格行内允许纵向扩展，由该行内容最高的卡片决定行高。板块说明移除现有的三行正文限制，使选择“完整展示”后不会继续静默截断主要判断。

```text
周期判断
┌────────────┬────────────┬────────────┐
│ 短期       │ 中期       │ 长期       │  同宽、同行等高
└────────────┴────────────┴────────────┘

板块判断
┌────────────┬────────────┬────────────┐
│ 板块 1     │ 板块 2     │ 板块 3     │  同宽、同行等高
├────────────┼────────────┼────────────┤
│ 板块 4     │ 板块 5     │ 板块 6     │
└────────────┴────────────┴────────────┘
```

### 视觉规则

沿用现有 `AppPalette`、`interactiveSurface`、颜色和字号，不新增视觉语言。此次签名变化是“报告式三栏节奏”：周期与板块沿同一列线对齐，让市场判断更像一张结构清晰的研究简报。

### 验证

- 先增加源码级回归断言，证明两个网格共用三列定义且卡片支持行内等高；旧实现应先触发失败。
- 运行趋势报告相关 XCTest，再运行完整 `swift test`。
- 构建 macOS App，并在最小宽度、常用宽度和截图中的宽屏宽度做人工检查：无横向溢出、无大块空列、同行卡片底边对齐、正文可见。

## 未采用方案

- **固定卡片高度并截断正文**：整齐但会隐藏分析依据，不符合已确认的可读性优先。
- **继续自适应列数，仅设置最小高度**：只能缓解高矮差，无法消除宽屏空列及周期/板块规格不一致。
