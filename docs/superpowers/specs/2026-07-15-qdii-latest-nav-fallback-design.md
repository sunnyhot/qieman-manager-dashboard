# QDII 最新净值兜底设计

## 背景与问题

个人持仓中的 `002286`、`100050`、`006327` 长期显示“—”。现场请求确认，这三个代码在盘中估值接口 `fundgz.1234567.com.cn/js/{code}.js` 上均返回空响应 `jsonpgz();`，但东方财富基金详情和最新净值接口仍有有效净值。

当前 Swift 原生链路为：加载完整历史净值，随后请求盘中估值；盘中估值为空时使用历史序列最后一条。完整历史请求或解析一旦失败，空历史仍会进入 12 小时缓存，之后的刷新会持续生成空报价。这三只基金没有盘中估值可掩盖该问题，因此稳定暴露为无数据。

## 目标

- 当盘中估值接口不覆盖某只基金时，仍展示最近一期官方单位净值、日期、市值和累计收益。
- 不让一次网络或解析失败造成最长 12 小时的持续空白。
- 保持现有持仓文件、SwiftUI 展示模型和正常基金的报价优先级不变。
- 使用确定性的模拟网络响应覆盖回归场景，不把真实第三方网络作为单元测试前提。

## 非目标

- 不为 QDII 计算自有盘中估值。
- 不改动用户持仓、成本或份额。
- 不调整页面布局、涨跌颜色或交互。
- 不重构平台调仓所需的完整历史净值链路。

## 方案选择

采用“盘中估值 → 轻量最新净值 → 完整历史净值”的三级兜底。

未采用的方案：

1. 仅缩短空缓存时间：可以减轻持续空白，但完整历史请求仍可能重复失败，无法稳定解决不提供盘中估值的基金。
2. 为三个代码配置固定映射或手工净值：短期可用，但不能覆盖以后出现的同类基金，维护成本和数据过期风险高。

## 数据源与解析契约

新增的内部请求为：

`GET https://api.fund.eastmoney.com/f10/lsjz?fundCode={code}&pageIndex=1&pageSize=1`

请求继续使用现有浏览器 User-Agent，并设置东方财富基金页面 Referer。只读取以下响应字段：

- `ErrCode`：为 `0` 才接受结果。
- `Data.LSJZList[0].DWJZ`：单位净值，必须能解析为大于零的 `Double`。
- `Data.LSJZList[0].FSRQ`：净值日期，允许缺失但优先保留。
- `Data.LSJZList[0].JZZZL`：最近净值日增长率；本次不映射到“今日涨跌”，避免把滞后的 QDII 净值变化误报成当日实时变化。

`Data`、`LSJZList`、首条记录或 `DWJZ` 缺失时，视为不可用并继续使用现有完整历史兜底。未知字段全部忽略。

## Swift 数据流

在 `QiemanPlatformNativeClient` 内保留现有 `NativeFundQuote` 输出，不新增 UI DTO：

1. 请求并解析盘中估值。存在官方净值或估值时保持当前行为。
2. 盘中估值为空时请求轻量最新净值，将结果映射为：
   - `price` / `officialNav` = `DWJZ`
   - `priceTime` / `officialNavDate` = `FSRQ`
   - `priceSource` = `official_nav`
   - `priceSourceLabel` = `最近净值`
   - `estimateChangePct` = `nil`，保留“最近净值”与“今日估值”的语义边界
3. 轻量接口不可用时，继续使用传入历史序列的最后一条。
4. 三个来源均不可用时才生成空报价。

为使测试可控，客户端增加默认值为 `URLSession.shared` 的会话注入点；生产行为不变，测试使用自定义 `URLProtocol` 返回固定响应。

## 缓存与错误处理

- 有效历史和有效报价沿用现有 TTL。
- 空历史不写入 12 小时缓存；后续刷新可以重新拉取。
- 空报价不写入正常报价缓存，避免 45 秒内继续复用已知不可用结果。
- 单个基金任一来源失败只降级该基金，不让整个持仓刷新失败。
- 不向日志写入 Cookie 或完整响应正文。

## 兼容性

这是内部兼容性增强：

- 持仓 JSON、SwiftUI 模型、页面字段和排序规则均不变。
- 正常返回盘中估值的基金不改变数据源优先级。
- 新增第三方读取接口属于外部依赖风险；通过保留完整历史兜底和宽松解析降低接口字段扩展或短时不可用的影响。
- 日期继续使用来源提供的 `yyyy-MM-dd` 文本，数值仍按现有展示层精度格式化。

## 测试与验收

新增 Swift 回归测试，模拟以下响应：

1. `fundgz` 返回 `jsonpgz();`，最新净值接口返回有效记录，断言持仓行具有市值、净值和日期，且不伪造今日涨跌率。
2. 最新净值响应缺少 `DWJZ` 或 `ErrCode` 非零，断言继续使用完整历史兜底。
3. 盘中估值有效，断言不会被最新净值覆盖。
4. 空报价和空历史不按正常 TTL 长时间复用。

实现后先运行新增测试，再运行 `swift test`，最后使用项目构建命令完成编译验证。验收标准是 `002286`、`100050`、`006327` 在只有最新官方净值可用时不再显示“—”，其他基金行为不变。
