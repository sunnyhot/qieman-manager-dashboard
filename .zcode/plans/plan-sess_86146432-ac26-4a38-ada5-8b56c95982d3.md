## 目标
重构趋势报告**整页布局骨架**。当前 6 模块单列纵叠成一条长龙，每个自带 trendReportSubHeader（标题+细线），无主次无分组节奏。改为**三分区聚拢**：①市场视图 ②操作建议 ③核验。

注：上一轮已把所有子卡片样式统一好（trendHorizonCard/trendSectorCard/trendAssetCard/trendActionCard/trendEvidenceCard/trendWarnings + 各 helper），**这一轮只改骨架，卡片本身不动**。

## 新骨架（trendReportBalancedLayout 重构）
```
VStack {
  trendPortfolioHeader          // 概览头（已有，精简）
  marketSection(report)         // ① 市场视图：周期 + 板块
  actionSection(report)         // ② 操作建议：重点标的 + 行动候选
  verificationSection(report)   // ③ 核验：证据 + 边界
}
```

### ① 市场视图（marketSection）
- 分区标题 trendReportSectionTitle（加粗 13pt + 品牌色小图标，比 subHeader 更有分量，作为分区锚点）
- 周期卡横向网格（minimum 200，复用 trendHorizonCard）
- 板块卡横向网格（minimum 200，复用 trendSectorCard）
- 周期/板块之间不加子标题（它们视觉上同属"市场"，直接两个网格块即可，用 spacing 分隔）

### ② 操作建议（actionSection）
- 分区标题
- 重点标的行卡（复用 trendAssetCard，单列）
- 行动候选行卡（复用 trendActionCard，单列）
- 两块间用 spacing 自然分隔

### ③ 核验（verificationSection）
- 分区标题
- 证据行卡（复用 trendEvidenceCard，单列）
- 边界与提示（复用 trendWarnings，含 disclaimer 底部）

## 关键改动
1. **新增** `trendReportSectionTitle(_ title:, icon:)`：分区级标题（13pt bold + 品牌色图标 + 底部细线），比 trendReportSubHeader 更有分量。保留 trendReportSubHeader（仍可能被内部用，不删）
2. **重写** `trendReportSectionGrid(_:)` → 拆为 `marketSection`/`actionSection`/`verificationSection` 三个方法。**保留 `trendReportSectionGrid` 标识符**（测试约束：改为调用三分区，或保留为兼容包装）—— 实测：测试断言 `trendReportSectionGrid` 存在于源码，我会让它仍存在（可在 trendReportBalancedLayout 里调用 trendReportSectionGrid，后者内部组织三分区）
3. `trendReportBalancedLayout` 改为 header + trendReportSectionGrid（内部三分区）
4. 6 个子模块的 *Grid/List 方法（trendHorizonGrid/trendSectorGrid/trendAssetList/trendActionList/trendEvidenceList/trendWarnings）**去掉它们各自的 trendReportSubHeader 调用**（标题上移到分区级，避免标题重复）
